Production Pipeline Controls

    policies:
      - name: error-traces
        type: status_code
      - name: high-latency
        type: latency
        latency: { threshold_ms: 1200 }

Risk Register

Top Operational Risks

HighSingle collector gateway

Failure can interrupt ingestion for all services.

Mitigation: active-active collectors + health-based failover.

MediumUnbounded log cardinality

Uncontrolled labels can spike cost and query latency.

Mitigation: enforce attribute policy and sampling controls.

MediumMissing ownership tags

Alerts lack direct team routing and increase MTTR.

Mitigation: block deploys without service/team metadata.

HighNo error budget policy

Feature velocity can erode reliability silently.

Mitigation: tie release gates to SLO burn trends.

Component Description Section

Describe Each Architecture Component

ADOT Collector

Role: Receive/process/export telemetry.

Why it exists: Decouple apps from backends.

Key config: `receivers`, `processors`, `exporters`.

CloudWatch

Role: Metrics, logs, alarms.

Why it exists: Native AWS observability baseline.

Key config: retention, metric filters, alarm policies.

X-Ray

Role: Trace visualization and dependency map.

Why it exists: Speed up root-cause isolation.

Key config: sampling rules, service naming, annotations.

AMP + Grafana

Role: Prometheus storage and dashboards.

Why it exists: Scalable metrics + cross-team insights.

Key config: scrape strategy, workspace IAM, dashboard folders.
